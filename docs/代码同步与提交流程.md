# NarratoAI：与上游同步 + 提交个人改动流程

本文档用于你当前仓库结构：
- `upstream`：原仓库（`https://github.com/linyqh/NarratoAI.git`）
- `origin`：你的 fork（`https://github.com/gavin99587/NarratoAI.git`）
- `main`：仅用于同步上游，不在此开发
- `feat/local-customizations`：你自己的长期开发分支

## 0. 首次检查（只需偶尔确认）

```bash
git remote -v
```

应看到：
- `origin` 指向你的仓库
- `upstream` 指向原仓库

如果缺少 `upstream`，执行：

```bash
git remote add upstream https://github.com/linyqh/NarratoAI.git
```

## 1. 同步上游最新代码到本地 main

```bash
git fetch upstream
git switch main
git reset --hard upstream/main
```

说明：
- 这会让本地 `main` 和上游 `main` 完全一致。
- 不要在 `main` 上写你自己的功能代码。

## 2. 把最新上游代码合并到你的开发分支

```bash
git switch feat/local-customizations
git rebase main
```

如果有冲突：

```bash
# 手动改冲突文件后
git add <冲突文件>
git rebase --continue
```

如果想放弃本次 rebase：

```bash
git rebase --abort
```

## 3. 日常开发与提交（在你的分支上）

```bash
git switch feat/local-customizations
git status
git add -A
git commit -m "feat: 你的改动说明"
```

## 4. 推送到你的 GitHub fork

```bash
git push origin feat/local-customizations
```

如果你刚做过 `rebase`，通常需要：

```bash
git push --force-with-lease origin feat/local-customizations
```

## 5. （可选）把同步后的 main 也推到 fork

仅在你希望 fork 的 `main` 也始终等于上游时执行：

```bash
git switch main
git push --force-with-lease origin main
```

## 6. 一套最常用命令（可直接复制）

```bash
git fetch upstream
git switch main
git reset --hard upstream/main
git switch feat/local-customizations
git rebase main
git push --force-with-lease origin feat/local-customizations
```

## 7. 推荐习惯

- 只在 `feat/*` 分支开发，不在 `main` 开发。
- 每次开始新功能前，先执行一次“第 1 + 第 2 步”。
- 提交信息建议用：`feat:`、`fix:`、`chore:` 前缀。
- 做高风险改动前，先建备份分支：

```bash
git switch feat/local-customizations
git switch -c backup/$(date +%Y%m%d-%H%M)-before-big-change
```
