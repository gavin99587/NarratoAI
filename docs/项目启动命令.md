# NarratoAI 启动命令

本文档记录本项目推荐启动方式（项目内隔离 HOME，避免首次 Streamlit 交互卡住）。

## 1. 本地运行（推荐）

### 1.1 进入项目目录

```bash
cd /Users/zhanghuiwen/webstorm_proj_demo/NarratoAI
```

### 1.2 创建并激活虚拟环境（首次）

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 1.3 安装依赖（首次或依赖变更后）

```bash
pip install -r requirements.txt
```

### 1.4 准备配置文件（首次）

```bash
cp config.example.toml config.toml
```

然后编辑 `config.toml`，填入你自己的模型/API Key 等配置。

### 1.5 推荐启动命令（隔离 HOME）

先初始化项目内 Streamlit 凭据（只需一次）：

```bash
mkdir -p .home/.streamlit
cat > .home/.streamlit/credentials.toml <<'TOML'
[general]
email = ""
TOML
```

再启动（建议单行，避免换行误操作）：

```bash
HOME=/Users/zhanghuiwen/webstorm_proj_demo/NarratoAI/.home streamlit run webui.py --server.maxUploadSize=2048 --server.port=8502
```

```bash
我自己用的快捷启动命令
cd /Users/zhanghuiwen/webstorm_proj_demo/NarratoAI
source .venv/bin/activate
HOME=/Users/zhanghuiwen/webstorm_proj_demo/NarratoAI/.home streamlit run webui.py --server.maxUploadSize=2048
--server.port=8502
```


说明：
- 使用项目内 `.home` 保存 Streamlit 配置，不污染全局环境。
- `credentials.toml` 为空邮箱可跳过首次 Email 交互提示。
- 可按需修改端口（如 `8503`、`8504`）。
- `--server.port=8502` 不要单独换到下一行，否则会被 shell 当作新命令。

## 2. Docker 运行

### 2.1 启动

```bash
docker compose up -d --build
```

### 2.2 查看日志

```bash
docker compose logs -f
```

### 2.3 停止

```bash
docker compose down
```

## 3. 常用维护命令

### 3.1 查看当前分支与状态

```bash
git branch -vv
git status -sb
```

### 3.2 查看端口占用（示例：8502）

```bash
lsof -i :8502
```

### 3.3 结束某个进程（把 PID 替换成真实值）

```bash
kill <PID>
```

## 4. 常见问题

- 启动报模块缺失：重新执行 `pip install -r requirements.txt`
- 端口被占用：换端口或先结束占用进程
- 配置错误：检查 `config.toml` 是否已正确填写
